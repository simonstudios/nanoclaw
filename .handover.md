## Current Problem: iCloud Drive Eviction

### Symptoms
Bot can `ls` files in `/workspace/extra/Documents` but **cannot read ~92% of them** — gets `Resource deadlock avoided` (EDEADLK).

### Root Cause (Diagnosed This Session)
- `~/Documents` is synced to iCloud Drive with "Optimise Mac Storage" enabled
- macOS evicts files not recently used — leaving only metadata stubs on disk
- These show as `compressed,dataless` flags in `ls -lO`
- Docker VirtioFS **cannot trigger iCloud downloads** — it sees the stub and deadlocks
- 7002 of 7564 files (92%) are evicted

### Evidence
- Fresh files / recently accessed files → ✅ readable from container
- Files with `compressed,dataless` flag → ❌ `Resource deadlock avoided`
- Running `cat file > /dev/null` on the host triggers iCloud download → then Docker can read it
- Initially misdiagnosed as quarantine xattr issue — that was wrong (both working and failing files had identical quarantine attributes; the differentiator is the `dataless` flag)

### Recommended Fix
**System Settings → Apple ID → iCloud → iCloud Drive → uncheck "Optimise Mac Storage"**

This tells macOS to keep all files downloaded locally. No code changes needed. The existing mount will just work.

## Key Architecture Notes

### Env Var Injection
- Per-group env file: `~/.config/nanoclaw/env/{group-folder}.env` (e.g., `main.env`)
- Lives on host, **never mounted** into containers — injected via `-e` flag at container start
- `readEnvFileFromPath()` in `src/env.ts` handles parsing
- `SECRET_ENV_VARS` in config — only for billing-sensitive keys (ANTHROPIC_API_KEY, CLAUDE_CODE_OAUTH_TOKEN)
- `GH_TOKEN` must NOT be in SECRET_ENV_VARS (gh CLI needs it in Bash)

### Mount Security
- Allowlist at `~/.config/nanoclaw/mount-allowlist.json` (outside project root, tamper-proof from containers)
- `src/mount-security.ts` validates mounts against allowlist + blocked patterns
- Default blocked patterns include `.ssh`, `.gnupg`, `.aws`, `.docker`, `.env`, credentials, etc.
- `nonMainReadOnly: true` forces read-only for non-main groups regardless of config
- Container mounts appear at `/workspace/extra/{containerPath}`
